How VibeDetect Works

⚡ Multi-Layer Analysis Pipeline

VibeDetect combines 5 distinct analysis layers: AST structural parsing, Shannon entropy calculation, pattern engine (regex + language-aware), statistical metrics, and ML model ensemble scoring. Each layer contributes weighted signals to the final verdict. Now powered by Claude AI for deeper semantic analysis.

🤖 AI Pattern Recognition

AI-generated code exhibits: perfect docstrings on all functions, overly descriptive variable names, consistent indentation uniformity, zero TODO/FIXME comments, symmetric error handling, comprehensive type annotations, and predictable structure. These are flagged as AI signals.

👤 Human Pattern Recognition

Human code shows: TODO/FIXME/HACK comments, abbreviations, inconsistent style, commented-out dead code, debugging artifacts, non-standard variable names, missing error handling in some places but detailed in others, and personal shortcuts.

📊 Entropy Analysis

Shannon entropy measures lexical diversity. AI code tends toward moderate-high entropy (predictable but varied vocabulary), while human code shows more extreme variance. Burstiness and token distribution are measured across 30 code segments to plot the entropy curve.

🧠 Claude AI Integration

Claude Sonnet performs deep semantic analysis: understanding code intent, naming patterns, comment quality, architectural decisions, and overall writing style. This adds a qualitative layer that pure statistical methods miss, significantly boosting detection accuracy.

🛡️ False Positive Mitigation

Language-aware analysis prevents cross-language false positives. Score normalization ensures consistent confidence ranges. Bayesian smoothing prevents overconfidence with few signals. Pattern overlap detection prevents double-counting. Minimum signal threshold required.
